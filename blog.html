Question-1: What are the different ways to select an element in the DOM?

Selecting elements is the first step in manipulating a webpage with JavaScript. The DOM (Document Object Model) provides several methods to find and select elements based on different criteria. Here are the main ways to select elements: 

              
                // Select by ID which returns a single element
                const header = document.getElementById('header');
                
                // Select by class name which returns a HTML Collection
                const paragraphs = document.getElementsByClassName('paragraph');
                
                // Select by tag name which returns a HTML Collection
                const divs = document.getElementsByTagName('div');
                
                // Select using CSS selectors
                const firstButton = document.querySelector('.btn');       // Returns first matching element
                const allButtons = document.querySelectorAll('.btn');     // Returns a List of all matching elements

I find querySelector and querySelectorAll to be the most flexible options since they accept any valid CSS selectors like tag, classname even element.

Question-2: What is the difference between innerHTML, innerText, and textContent?

When working with text content in the DOM, JavaScript gives us three similar looking properties those are innerHTML, innerText, and textContent. Though they might seem interchangeable at first but they each have unique behaviors which can be useful for different situations.

  • innerHTML: Returns the complete HTML inside the element, including all tags. It can also be used to change the content of an element.
  • innerText: Returns only the visible text content of the node and its descendants. It can also be used to change the content of an element. It ignores hidden elements like display: none.
  • textContent: Returns all text content, including hidden texts. It can also be used to change the content of an element. But it treats HTML tags as plain text when setting content which means it will not render the HTML tags.

Question-3: What is event delegation in the DOM?

Event delegation is a technique where instead of attaching event listeners to individual elements, we attach a single event listener to a parent element. This parent listener then "delegates" the handling of events to the appropriate child elements based on which one triggered the event. 

            
              // Without event delegation (attaching listeners to each button)
              document.querySelectorAll('.button').forEach(button => {
                button.addEventListener('click', function() {
                  console.log('Button clicked:', this.textContent);
                });
              });

              // With event delegation (single listener on parent)
              document.querySelector('.task-card').addEventListener('click', function(event) {
                if (event.target.classList.contains('button')) {
                  console.log('Button clicked:', event.target.textContent);
                }
              });

In assignment I've had used the first approach to attach listeners to each button. But the second approach is more efficient and recommended when working with a large number of elements. But after getting the idea of event delegation with this question I've changed the approach to the second one (event delegation) in my assignment.

Question-4: What is event bubbling in the DOM?

Event bubbling is the way an event travels up through the DOM hierarchy after it's triggered. When an event occurs on an element (like a click), it first runs the handlers on that element, then on its parent, then all the way up through its ancestors. This bubbling behavior is the default way events work in the DOM. It allows us to listen for events on parent elements and have them be triggered by their children.

Question-5: How do you create, add, and remove elements using JavaScript?

When we want to create new elements for our webpage using JavaScript, we're essentially crafting pieces of our interface dynamically rather than having them pre-written in HTML.

  • The first step is creation. We use the createElement() method, which is like getting a blank canvas. But this element is not yet a part of the DOM. It's just a JavaScript object.
  • The next step is to add the element to the DOM. We use the appendChild() method to add the element as a child of another element. This is how we make our element visible on the page.
  • Finally, if we want to remove an element from the DOM, we use the removeChild() method. This method removes a specified child node of the specified element.
